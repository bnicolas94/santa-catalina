Resumen EjecutivoEl Excel analizado (11111.xlsx) contiene 5.212 pedidos registrados durante 2026 en la hoja '2026', mas tres hojas de ruta diaria (manana/siesta/tarde) que se arman manualmente cada dia para los choferes. El objetivo es eliminar la carga manual duplicada: hoy el flujo es WhatsApp → Excel → sistema, y se busca reducirlo a WhatsApp → sistema directamente, o como minimo a Excel → sistema con una importacion masiva.Este documento detalla: (1) la estructura del Excel, (2) los gaps entre el Excel y el modelo de datos del sistema, (3) la estrategia de migracion recomendada, y (4) las tareas concretas a implementar.2. Estrategia de Integracion Recomendada4.1 Flujo PropuestoFASE 1 — Migracion historica (one-shot): Importar todos los pedidos de 2026 al sistema. Objetivo: tener el historico en la base de datos para reportes y referencias.FASE 2 — Integracion operativa diaria: El Excel se sigue usando como entrada, pero se agrega un boton 'Importar al sistema' que procesa las filas nuevas del dia automaticamente.FASE 3 — Reemplazar el Excel (opcional, largo plazo): El equipo carga pedidos directamente en el sistema desde el celular, eliminando el Excel. Solo viable cuando el sistema sea lo suficientemente agil para el ritmo de pedidos por WhatsApp.4.2 Propuesta Tecnica: Endpoint de Importacion MasivaLa solucion mas limpia para el sistema Next.js existente es crear un nuevo endpoint y pagina de importacion:ComponenteDescripcionPOST /api/importar-pedidosRecibe un array de pedidos parseados desde el Excel. Recomendacion FinalEl camino mas pragmatico, considerando que el Excel es la herramienta operativa actual y que hay un sistema funcional construido:Implementar primero el endpoint de importacion masiva (tareas 1-6) sin tocar la UI del Excel existente. Esto permite migrar el historico de 2026 y tener una base de datos limpia.Agregar la pagina /importar al dashboard (tareas 9-13). El flujo diario pasa a ser: Excel → subir al sistema al inicio del dia → gestionar desde el sistema.Con el sistema poblado de datos reales, identificar friccion: que es tedioso ingresar en el sistema que en Excel. Iterar la UI del modulo de pedidos y rutas para igualarlo.Solo cuando el sistema sea mas rapido que el Excel para el operador, eliminar el Excel del flujo diario.Documento preparado para Antigravity — Santa Catalina 2026
